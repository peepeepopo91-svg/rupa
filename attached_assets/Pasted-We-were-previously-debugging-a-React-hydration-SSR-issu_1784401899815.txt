## What was identified

The root cause appears to be in **`Footer.tsx`**.

It currently does something similar to:

```tsx
useState(getSiteContent)
```

This passes `getSiteContent` as a lazy initializer.

React executes that function during:

* Server-Side Rendering (SSR)
* Client hydration

The problem is that the function returns **different values** in each environment:

* During SSR there is no `localStorage`, so it returns the default content.
* During client hydration it reads the admin-saved values from `localStorage`.

This mismatch causes React hydration to fail, which then cascades into the **Invalid Hook Call** errors.

The previous AI's planned fix was:

* Export the default content as a constant (for example `CONTENT_DEFAULTS`) from the store.
* Use those defaults as the initial state so SSR and the first client render are identical.
* Only read from `localStorage` after hydration (inside an effect or another client-only mechanism).
