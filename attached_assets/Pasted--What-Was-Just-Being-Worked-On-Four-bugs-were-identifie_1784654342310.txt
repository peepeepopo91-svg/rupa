# Bug 1 — 9 Teams Registered but Only 8 Show in the Bracket

## Root Cause

`getMs()` in:

```

src/components/tournament/TournamentBracket.tsx

````

filters out **all** bye matches:

```ts
.filter((m): m is Match => !!m && m.status !== 'bye')
````

With 9 teams, the bracket pads to 16 slots.

Team 9 receives a bye (auto-advance), but because the bye match is filtered out entirely, Team 9 never appears in the bracket.

## Fix

Include bye matches that have a real team advancing (`winnerId !== null`):

```ts
.filter((m): m is Match =>
  !!m && (m.status !== 'bye' || m.winnerId !== null)
)
```

Then, inside `MatchCard`:

* If `match.status === 'bye'`
* Render a special slim card
* Show only the advancing team
* Display an **"AUTO BYE →"** badge
* Do **not** render it like a normal match card

Apply the **same fix** to `getMs()` in:

```
src/components/admin/AdminBracketEditor.tsx
```

# Bug 4 — Right Side of Bracket Overflows Outside the Container

## Root Cause

Inside `BracketView`:

```ts
const naturalW =
  leftW + finalsW + rightW
```

This calculation does **not** include the outer padding:

```
OUTER_PAD * 2
```

(24px × 2 = 48px)

The inner flex container uses that padding, so the actual content becomes wider than `naturalW`.

As a result:

* Scaling is incorrect.
* The right-side bracket extends outside the container.

## Fix

Change:

```ts
const naturalW =
  leftW + finalsW + rightW
```

to:

```ts
const naturalW =
  leftW + finalsW + rightW + OUTER_PAD * 2
```
