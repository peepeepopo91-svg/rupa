The GitHub Sync / Push system is broken and needs to be fixed completely.

The system is NOT pushing all changed files to GitHub. I discovered that 52 files were pending to commit, but the GitHub Sync Center failed to detect or push them. I had to manually fix everything through the terminal, which caused many Git errors, rebase conflicts, and data corruption issues.

The biggest problem is data/players.json. The sync system is not handling Git conflicts correctly and is allowing merge conflict markers and duplicated JSON data to enter the file.

The expected behavior:
- Clicking "Push to GitHub" should detect every changed, deleted, renamed, and untracked file.
- It should commit all pending files.
- It should validate important data files before pushing.
- It should safely handle Git divergence and conflicts.
- It should never corrupt JSON files.
