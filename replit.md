# Blue Tiers

A Minecraft PvP tier list website built with TanStack Start (React SSR), TanStack Router (file-based routing), Tailwind CSS v4, and Vite.

## How to run

```
npm run dev        # dev server (port 5000, hot-reload)
npm run build      # production build → dist/
npm run start      # serve the production build
```

The **Start application** workflow runs `npm run dev` automatically.

## Stack

| Layer | Technology |
|-------|------------|
| Framework | TanStack Start v1 (SSR) |
| Frontend | React 19, TanStack Router v1 |
| Build | Vite 7 |
| Styling | Tailwind CSS 4 |
| Language | TypeScript 5 |
| Deployment | Replit (autoscale) |

## Project structure

```
src/
├── components/       # UI components (Navbar, Hero, PlayerCard, shop/, admin/)
├── data/             # Types & constants (players, mining, shop, etc.)
├── routes/           # File-based routes (index, rankings, mining, exchange, shop, admin)
├── server/           # Server functions (auth, adminAuth, miningServer, shopServer, sseRegistry)
├── store/            # Client state stores (miningStore, shopStore, syncStore, etc.)
├── context/          # React contexts (MiningContext — owns the SSE connection)
├── routeTree.gen.ts  # Auto-generated by TanStack Router — do not edit
├── router.tsx        # Router factory
└── styles.css        # Global CSS, Tailwind import, animations

data/
├── players.json          # Tier list players
├── mining-users.json     # Mining accounts & balances (including gems)
├── mining-community.json # Community mining state
├── shop-items.json       # Shop catalog (editable by admin without code changes)
├── shop-purchases.json   # All purchase records
├── economy.json          # Economy overrides
├── content.json          # Site content
└── event.json            # Event config
```

## Key files

- `src/data/players.ts` — single source of truth for all player data
- `admin.yml` — admin credentials (username, password1, password2)
- `credentials.yml` — user credentials

## Replit configuration

- `vite.config.ts` has `server: { host: true, allowedHosts: true }` for Replit's proxied preview
- Dev port changed from 3000 → 5000 to match Replit's webview requirement
- Netlify plugin removed; production build uses `server.mjs` (srvx/node) to serve `dist/server/server.js` + static assets from `dist/client/`
- Deployment configured as autoscale: build = `npm run build`, run = `node server.mjs`
